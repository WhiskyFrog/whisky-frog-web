# Build the structured generic facet renderer and preservation reducer

## Objective

Replace the hardcoded flat-response sidebar internals with a generic renderer
over the deployed structured response union and a pure selection reducer. Use
the generated discriminator exhaustively to choose controls, and use
server-supplied labels, query metadata, `selection_mode`, `selected`,
`relevant`, bounds, units, and dependency parents for content and behavior.
Support explicit `peated_state` values including `unknown` and the complete
server-provided `edition_state` domain. Preserve selected values when their
count becomes zero or when a dependency makes them irrelevant until the user
clears that value or invokes reset. Retain and complete the responsive drawer's
keyboard and accessibility behavior without fetching page data in this Task.

This Task requires approval because it replaces the shared filter interaction
surface used by both public product experiences.

## Acceptance Criteria

- The renderer switches exhaustively on the generated response discriminator
  and renders every variant present in the deployed contract. Adding an
  unhandled generated variant produces a type/test failure; no facet-key list
  chooses labels, control types, query names, selection modes, bounds, units,
  or dependency layout.
- Facet headings, option text, units, input names/values, selection behavior,
  and dependency-parent association come from server fields. A fixture with
  unfamiliar but valid labels/keys renders correctly without a component code
  change, proving the renderer is generic rather than a renamed hardcoded map.
- The pure reducer reconciles new server responses with URL/query state without
  dropping selections. Tests prove that a selected option with count `0`
  remains visible and selected, a selected facet with `relevant: false`
  remains represented and explicitly clearable, and selected numeric `0` and
  boolean `false` survive response refreshes and serialization. Only an
  individual clear action or the explicit reset removes them.
- `peated_state` renders and round-trips distinct peated, unpeated, and unknown
  choices. Every `edition_state` option supplied by the fixture renders and
  round-trips; no edition list is embedded in frontend code.
- Single-select, multi-select, bounded/unbounded range, and dependency-parent
  fixtures produce the contract-appropriate native controls and enforce the
  supplied selection mode. Bounds and units are presented without overriding a
  selected out-of-current-range value before the user edits or clears it.
- Dependency rendering keeps parents before/associated with children, conveys
  relevant/irrelevant state without making a retained selected child
  unreachable, and never clears a child as a side effect of changing a parent.
- Component tests cover option selection/deselection, radio/single replacement,
  range edits, individual clear, reset, zero-count preservation, irrelevant
  preservation, peated unknown, edition state, server-label substitution, and
  dependency-parent updates with the real normalized serializer boundary.
- The drawer retains backdrop close, Escape close, `inert`/`aria-hidden`, and
  body-scroll restoration, and adds deterministic focus entry, focus
  containment, trigger-focus restoration, and accessible dialog labeling.
  All controls have programmatic names/groups, state is not conveyed by color
  alone, and component tests exercise keyboard-only open, navigate, change,
  clear, and close flows at narrow and wide viewport classes.
- An automated accessibility assertion over representative fixtures reports no
  serious violations, and the component/query test commands plus
  `npx tsc --noEmit` pass.

## Constraints

- Do not fetch lists/facets or own catalog/per-market count wording in this
  component; accept structured data, normalized state, and scope copy via props.
- Do not recreate the old `axes` checks or a switch keyed by known domain names
  under the generic renderer.
- Do not silently discard unknown discriminator data at runtime. Contract drift
  must be observable through the exhaustive check and a clear safe error state.
- Do not change product cards, market tables, pagination, list search, price
  calculations, or the price-history endpoint/component in this Task.
- Do not change whisky-frog-lab, deploy, mutate production, or contact billable
  providers.

## Context

`ProductFacetSidebar.tsx` currently owns a fixed `ProductFilters` shape and
hardcodes every axis, label, option layout, boolean peated buttons, numeric
limits, and Korean units. Its drawer already provides useful Escape, backdrop,
inert, and scroll-lock behavior, but selected values only exist in local state
and panels can disappear with `axes` or empty option arrays. A generated-union
renderer and reducer are needed to consume structured v2 semantics safely.
